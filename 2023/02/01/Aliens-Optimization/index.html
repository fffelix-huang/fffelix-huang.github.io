Aliens 優化應用

基本概念

有些題目要求進行 $k$ 次操作的最大/最小值。我們開心的列出 $dp[i][j]$ 為前 $i$ 個東西進行 $j$ 次操作的最大值，答案就是 $dp[n][k]$，但很不幸的 $O(nk)$ 會爆，因此我們要換個做法。

考慮一個函數 $f(k)$ 為進行 $k$ 次操作的最大值。$f(k)$ 有一個關鍵的性質是他必須要是一個凹函數 (差分非嚴格遞減) 才能套用 Aliens 優化，也就是說每多進行一次操作，數值增加的幅度會越來越少 ($f(k + 1) - f(k) \leq f(k) - f(k - 1)$)。

我們定義另外一個函數 $g_p(k) = f(k) - kp$，也就是對於每筆操作，我們額外收取 $p$ 元的手續費。不難看出 $g_p(k)$ 也會是一個凹函數，而且當 $p$ 越大，$\max g_p(k)$ 極值發生的位置 $k$ 會越往左邊靠 (可以想成手續費太貴，所以不進行那麼多次操作)，當 $p$ 越小，極值發生的位置會越往右靠 (手續費太便宜，多進行幾次操作不會虧錢)。[1]

有了這些概念，我們就可以對 $p$ 進行二分搜，讓 $g_p(k)$ 極值發生的位置剛好落在 $k$，也就是操作 $k$ 次，這樣我們就可以算出 $f(k) = g_p(k) + kp$ 了！要找最小值的話就改成 $g_p(k) = f(k) + kp$，答案是 $f(k) = g_p(k) - kp$，也能用同樣的邏輯。

其他的細節可以參考 Reference 的文章。

模板

找最小值的話記得把 $+kp$ 改成 $-kp$。

例題

底下的例題 $dp$ 應該要是 {最大/最小值, 最少操作次數}，方便起見只寫最大/最小值，實作時記得要計算操作次數。

AI-666 賺多少

已知 $n$ 個時間點股票的價格，手上沒有股票的話才能買入。求買入 $k$ 次的最大利益？

本題有 greedy 解，但我們練習用 Aliens 優化來做。

$f(k)$ 是凹函數，因為如果第 $k$ 筆交易的獲益比第 $k - 1$ 次多，我們可以交換交易的順序，把 $k - 1$ 多做的那次換成 $k$ 多做的那次。

定義 $dp$：

  • $dp_0[i]$ = 時間 $i$ 時手上沒有股票的最大收益
  • $dp_1[i]$ = 時間 $i$ 時手上持有股票的最大收益

轉移就會是：

\begin{aligned}
dp_0[i] &= \max(dp_0[i - 1], dp_1[i - 1] + a[i] - p) \\
dp_1[i] &= \max(dp_1[i - 1], dp_0[i - 1] - a[i])
\end{aligned}

計算一次 $dp$ 的時間為 $O(n)$，因此總時間複雜度為 $O(n \log C)$ (C 是 Aliens 二分搜的範圍)。
